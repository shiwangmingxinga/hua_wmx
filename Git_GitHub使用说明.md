# Git 和 GitHub 有什么用？

## 🎯 一句话总结

- **Git** = 帮你记录代码的每一次修改（本地备份）
- **GitHub** = 把代码存到网上（云端备份 + 分享）

---

## 📚 二、什么是Git？

### 1.1 Git是什么？
Git是一个**版本控制工具**，就像游戏的存档功能。

### 1.2 为什么要用Git？
| 不用Git | 用Git |
|---------|-------|
| ❌ 改坏代码找不回来 | ✅ 随时可以回退到之前版本 |
| ❌ 文件名乱糟糟：`代码_v1`, `代码_v2_final` | ✅ 清晰记录每个版本的改动 |
| ❌ 多人协作容易覆盖别人的代码 | ✅ 不会丢失任何人的代码 |
| ❌ 误删文件找不回 | ✅ 所有历史都有记录 |

### 1.3 Git的核心概念
```
工作区 → 暂存区 → 本地仓库 → 远程仓库
```

- **工作区**：你正在编辑的代码文件
- **暂存区**（Stage）：准备保存的修改
- **本地仓库**：已保存的版本记录
- **远程仓库**：GitHub等云端服务器

---

## 🌐 三、什么是GitHub？

### 3.1 GitHub是什么？
GitHub是一个**代码托管平台**，相当于代码的"网盘"。

### 3.2 GitHub能做什么？
| 功能 | 说明 |
|------|------|
| 📦 云端备份 | 代码不会因电脑损坏丢失 |
| 🔄 版本同步 | 在不同电脑上都能访问最新代码 |
| 🤝 团队协作 | 多人可以同时开发同一项目 |
| 🌍 开源分享 | 可以看到和学习别人的项目 |
| 📝 简历加分 | 展示你的代码作品 |

---

## 🔄 四、Git和GitHub的关系

```
┌─────────────────────────────────────────────────┐
│  你（本地电脑）           GitHub（云端）        │
│                                                 │
│  写代码                  备份代码               │
│    ↓                      ↑                     │
│  git add .   ──push──→   远程仓库              │
│    ↑                      ↓                     │
│  git commit   ←pull──    其他人改的代码         │
│    ↑                                            │
│  git log       ←查看历史记录→                  │
└─────────────────────────────────────────────────┘
```

## 📝 五、日常使用流程

### 5.1 第一次设置（只需做一次）
```bash
# 1. 配置你的身份
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"

# 2. 进入项目文件夹
cd e:\你的项目文件夹

# 3. 初始化Git仓库
git init

# 4. 关联GitHub仓库
git remote add origin https://github.com/你的用户名/仓库名.git
```

### 5.2 每次保存代码
```bash
# 1. 添加所有修改
git add .

# 2. 保存当前版本（写清楚你做了什么）
git commit -m "修复了登录页面的bug"

# 3. 推送到GitHub
git push
```

### 5.3 如果在其他电脑继续开发
```bash
# 1. 下载最新代码
git pull

# 2. 然后正常add、commit、push
```

## 💡 六、为什么要用它们？

### 6.1 防止代码丢失
```
电脑坏了？硬盘坏了？误删了？
→ 不怕！代码在GitHub上！
```

### 6.2 追踪修改历史
```
想知道上周改了什么？
→ git log 一目了然
→ 随时可以回退到任意版本
```

### 6.3 多人协作
```
小王改了一版，小李改了一版
→ 不会覆盖！
→ 每个人都有自己的代码记录
```

### 6.4 展示作品
```
面试时：看我GitHub上有20个项目
→ 证明你有实战经验！
```

## 🎓 七、常见命令速查

| 命令 | 作用 |
|------|------|
| `git init` | 初始化新仓库 |
| `git add .` | 添加所有修改 |
| `git commit -m "说明"` | 保存当前版本 |
| `git push` | 推送到GitHub |
| `git pull` | 从GitHub拉取最新代码 |
| `git status` | 查看当前状态 |
| `git log` | 查看历史记录 |
| `git diff` | 查看具体改了什么 |

---

## ⚠️ 八、注意事项

1. **每次重要修改后记得commit** - 不要攒太多修改一起提交
2. **commit描述要写清楚** - 方便以后查找
3. **重要文件不要提交** - 比如密码、密钥等（可以用.gitignore忽略）
4. **网络不好可以重试** - GitHub访问有时不稳定是正常的

---

## 🎯 总结

> **Git = 时光机（记录每次修改）**
> **GitHub = 云端备份（代码永不失联）**

学会用它们，你的代码就安全了！🚀
